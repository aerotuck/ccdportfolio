Introduction

In this post we introduce the American Statistical Association’s (ASA) Committee on Career Developments (CCD) Portfolio project. The goal of this project is to make it easier for people to build their own personal portfolio website. In this project we will lay out and explain the installation of an opinionated set of technology tools, as well as some advice in the best ways to effectively communicate technical skills in such a website. Additionally, we will provide some advice on how to use the website to add to your personal brand and link posts to social media to build a reputation.

Why should you build a personal portfolio website? There are a few reasons:

  1. To communicate your diverse technical skills to potential employers or graduate advisers in a more thoughtful, comprehensive way than is possible through a typical interview process. A well crafted portfolio site not only allows one to demonstrate statistical and programming skills, but also the ability to effectively communicate.
  2. As a way to document for work that you have done in a way that is both easy for you to find to remind yourself of the work or to share out to others at any date
  3. As a way to help find individuals with similar research interests
  4. To learn how to build a website
  5. To give opportunities for the unexpected

Technologies we will be using and promoting

There is a very wide variety of potential blogging and website platforms, but the focus of the project is to be able to build a simple framework to communicate statistical and statistical programming abilities and not web development, so we chose a set of technologies that will make this the easiest. For reference, all of these tools are being used for this website, and most certainly for this post. The tools that we will be focused on here are:

  1. R (though some posts may use Python). It’s free and open sources, so that readers of your blog posts can actually try out the code that you are using without needing to pay for a license. It has an enormous set of statistical packages. Many statisticians use it, so the code won’t be foreign for a large number of users. We will be pretty agnostic on the text editor or Integrated Development Environment, though RStudio is recommended, though any variety of choices here are perfectly great.
  2. Blogdown This R package makes it easy to create blog websites using Rmarkdown and Hugo.
  3. Hugo This Go language based framework is the target from blogdown. It build fast, static websites that support a fair bit of customization, has a lot of user-built themes and is very customizable.
  4. Netlify This company provides a very easy and (for low volume sites) free hosting and deployment.
  5. Github Using Github to host your blog code, Netlify will automatically detect changes on the master branch, build and deploy the updated website very, very quickly.

Initial steps

We will release a succession of blogpost in the coming weeks, months, and years. In the near term, we will focus on the initial installation process to get your site up and going. From there we will provide some examples, tips, guest posts etc to help you make the most out of your Portfolio site!
